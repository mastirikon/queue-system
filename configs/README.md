# Конфигурация

## Переменные окружения

Все настройки загружаются из переменных окружения.

### Общие

- `ENV` — окружение (development/production), по умолчанию: `development`

### API сервис

- `API_PORT` — порт HTTP сервера, по умолчанию: `8080`
- `API_HOST` — хост для прослушивания, по умолчанию: `0.0.0.0`
- `API_READ_TIMEOUT` — таймаут чтения запроса, по умолчанию: `10s`
- `API_WRITE_TIMEOUT` — таймаут записи ответа, по умолчанию: `10s`
- `API_SHUTDOWN_TIMEOUT` — таймаут graceful shutdown, по умолчанию: `30s`

### Worker сервис

- `WORKER_CONCURRENCY` — количество одновременно обрабатываемых задач, по умолчанию: `10`
- `WORKER_RETRY_INTERVAL` — интервал между retry, по умолчанию: `10s`
- `WORKER_MAX_RETRIES` — максимальное количество попыток (8640 = 24 часа при 10 сек), по умолчанию: `8640`
- `WORKER_REQUEST_TIMEOUT` — таймаут HTTP запроса к внешнему API, по умолчанию: `30s`

### Redis

- `REDIS_ADDR` — адрес Redis сервера, по умолчанию: `localhost:6379`
- `REDIS_PASSWORD` — пароль (если требуется), по умолчанию: пусто
- `REDIS_DB` — номер базы данных, по умолчанию: `0`

## Использование

### Development

Создай файл `.env` в корне проекта (он в `.gitignore`):

\`\`\`bash
cp configs/.env.example .env
# Отредактируй .env под свои нужды
\`\`\`

### Production

Используй переменные окружения напрямую (Docker, K8s, systemd и т.д.).
